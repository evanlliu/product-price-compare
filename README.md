# 商品比价器 Price Compare v1.0.50

本版本基于 **v1.0.49** 继续优化，重点修复“点击识别没反应 / 纯链接识别不清楚”的问题，并把智能粘贴拆成两层逻辑：

1. **分享文案识别**：如果淘宝 / 抖音 / 京东分享文案里本身带了商品标题、价格、规格，前端会直接从文字里识别。
2. **纯链接识别**：如果只有链接，前端会通过 Cloudflare Worker 尝试打开链接、跟随跳转，并读取页面标题 / 商品 ID / 可能出现的价格或规格。

> 注意：很多平台不会在纯链接或公开页面里直接暴露实时价格，尤其是需要登录、App 内打开、活动价、券后价、会员价等情况，所以纯链接通常无法 100% 自动拿到真实价格。识别不到时会明确提示，需要手动补充金额和规格。

## v1.0.50 更新内容

1. 修复智能粘贴识别体验。
   - 点击“识别”后会立即显示识别结果。
   - 如果配置了 Cloudflare Worker，会继续尝试解析链接。
   - 如果 Worker 无法拿到价格，会在识别说明里提示，不再像“没反应”。

2. Cloudflare Worker 新增链接解析接口。
   - 新增 `?action=parseProductLink`。
   - 用于解析淘宝 / 抖音 / 拼多多 / 京东分享链接。
   - 会尝试获取最终跳转地址、页面标题、商品 ID、价格、规格。

3. 识别结果新增显示。
   - 商品 ID。
   - 最终跳转链接。
   - 识别说明。

4. 生成价格记录时会重新执行一次识别。
   - 避免只是输入框预览，没有真正调用 Worker。

5. 同步状态时间格式保持为 `HH:mm:ss`。

## GitHub 需要更新

需要上传 / 覆盖：

- `index.html`
- `README.md`
- `worker.js`

通常保留线上已有的：

- `data.json`

不要用压缩包里的示例 `data.json` 覆盖线上真实数据，除非你确认要重置数据。

## Cloudflare 需要更新

需要把新版 `worker.js` 复制到 Cloudflare Worker 中并部署。

Cloudflare 变量继续使用原来的：

- `APP_PASSWORD`
- `DATA_PATH`
- `GH_BRANCH`
- `GH_OWNER`
- `GH_REPO`
- `GH_TOKEN`

## 使用说明

### 分享文案

例如淘宝 / 抖音 / 京东这类带商品标题的分享文案，识别效果最好，因为标题本来就在文案里。

### 纯链接

例如：

```text
https://mobile.yangkeduo.com/goods1.html?ps=gXtuMvBLWH
```

这种链接本身通常不包含商品名称和价格，所以系统只能尝试通过 Worker 打开链接后解析页面。是否能拿到标题、价格，取决于平台页面是否公开这些信息。
